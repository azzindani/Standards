# MCP Server Ideas — Complete Reference

> All servers follow the General-Purpose MCP Server Development Standards.
> Two categories: **Local Execution** (self-hosted, offline-capable) and **API-Based** (public data feeds, no user data sent).
> Every server: 5–10 tools, inference loop capable, pipeline-ready.

---

## Legend

| Field | Description |
|---|---|
| **#** | Global index |
| **Server** | snake_case server name (matches directory naming convention) |
| **Category** | Execution model: Local or API-Based |
| **Lang** | Primary implementation language(s) |
| **Tools** | Tool count (5–10) |
| **Core Libraries** | Key local libraries or public APIs used |
| **Bulk Input** | What the server can ingest in bulk |
| **Pipeline Stages** | High-level pipeline flow |
| **Internal Complexity** | Engine-level logic patterns |
| **Inference Loop Trigger** | What forces the LLM to reason between tool calls |
| **Replaces** | Cloud service or paid tool this eliminates |
| **Privacy Benefit** | Why local/public-only matters for this use case |

## Part 1 — Local Execution Servers

All servers in Part 1 run entirely on local hardware. No user data leaves the machine. Internet connection required only during installation for dependency downloads.
